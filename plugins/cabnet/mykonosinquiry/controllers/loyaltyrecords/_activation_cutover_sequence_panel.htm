Activation cutover sequence

Before Loyalty Continuity becomes a live operational surface, the first activation should follow a strict cutover order: confirm storage installation readiness, validate guarded toolbar behavior, verify that Inquiry Queue remains the system of action, nominate the first pilot owner, define the first approved cohort, and require a rollback path before any live continuity record is created.

  1. Freeze the first pilot scope to a narrow, approved candidate lane.
  2. Confirm suppression, consent, exception-routing, and escalation rules are already documented.
  3. Install loyalty storage only in a dedicated structural release.
  4. Verify create, list, and update pages render normally with real storage.
  5. Move one controlled cohort from planning mode to live continuity handling.
  6. Review outcomes against rollback criteria before broadening use.
