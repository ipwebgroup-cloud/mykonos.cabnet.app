Touchpoint ledger packet

Loyalty should not only remember that a record moved into continuity; it should preserve each meaningful retention touchpoint as a disciplined ledger. This packet stages the storage structure for those touchpoints without forcing live activation yet.

  • Anchor: loyalty record id, source inquiry id, and reference code
  • Event type: review, outreach, revisit prompt, referral nurture, or return-value follow-up
  • Channel + outcome: contact route used and the operating result that followed
  • Timing discipline: touchpoint date and optional next-step date
  • Summary payload: operator summary and compact JSON carryover for continuity evidence
