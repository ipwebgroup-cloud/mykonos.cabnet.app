Exception routing

Not every promising record belongs in Loyalty Continuity. Define exception-routing rules now so cases that still need active handling, recovery, or operator review do not get pushed into a retention lane prematurely.

  • Open-service exception: if service delivery is still active, keep the record in Inquiry Queue.
  • Recovery exception: if trust repair or issue resolution is unresolved, route to operator review first.
  • Ambiguous-value exception: if return-value is unclear, do not classify into a live pilot yet.
  • Channel-risk exception: if the safe outreach route is uncertain, delay continuity activation.
  • Escalation anchor: when in doubt, preserve premium handling and route back to the live concierge workspace.
