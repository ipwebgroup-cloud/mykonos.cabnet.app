Dependency map

Loyalty Continuity should only move from planning mode to live activation when the surrounding dependencies are visibly aligned. This dependency map keeps operators focused on the real prerequisites: inquiry handoff discipline, ownership clarity, approved communication channels, consent discipline, suppression controls, and post-launch review coverage.

  • Inquiry Queue dependency: active concierge handling must remain the live source of action until loyalty storage is installed.
  • Operator dependency: an explicit owner, escalation path, and fallback owner should already be nominated.
  • Communication dependency: channel priority, message framework, and frequency-cap rules should already be approved.
  • Control dependency: consent, suppression, exception routing, and rollback criteria must already be documented.
  • Review dependency: measurement, return-value signals, and post-launch review checkpoints must be in place before scale-up.
