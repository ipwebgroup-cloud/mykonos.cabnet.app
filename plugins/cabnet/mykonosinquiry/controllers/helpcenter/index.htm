Inquiry Queue

Use the queue to read live operational posture before drilling into record detail. The queue should stay scan-friendly: owner state, urgency, linked continuity, latest packet signal, and next review timing should be visible before an operator opens deeper panels.

Queue reading order
Scan owner, urgency, and follow-up first
Start with who owns the inquiry, whether follow-up is overdue or due today, and whether the guest still needs first-touch or proposal shaping before loyalty work begins.
Linked continuity
Use packet and next-review cues before opening history
Linked queue rows should let operators read the latest packet posture and revisit timing at a glance. Open loyalty history only when the compact row cue shows something that needs deeper context.
Filter discipline
Use the Loyalty Posture filter to isolate transfer work
Separate linked, transfer-ready, draft-ready, and queue-only rows so continuity work does not widen the whole queue. This keeps proposal shaping and continuity stewardship from competing visually.

Loyalty Continuity

Loyalty Continuity exists to preserve remembered-return value, referral potential, and revisit timing without confusing draft records with active retention work.

List posture
Use the list for latest outcome, latest packet, and next review
The list should stay compact and scan-friendly. Operators should see the source inquiry backlink, continuity posture, and next review cue before opening a record.
Create mode safety
Keep inquiry-backed drafts distinct from manual drafts
If a loyalty record came from a real inquiry, keep the source reference visible. If it is a manual draft, keep that distinction explicit so operators do not mistake it for a fully linked continuity lane.
Workspace tabs
Compare source posture before editing continuity fields
Overview, Workspace, and History should all let operators compare source inquiry posture against current continuity posture before they widen retention notes or review packet strategy.

Queue-to-loyalty bridge

Use continuity only when it helps operators route real post-inquiry value. Closed or maturing records can move into loyalty, but the queue should still show whether that handoff already happened.

Linked to loyalty
Continuity already owns the remembered-return lane
The source inquiry already has a loyalty record. Operators can open the linked record or jump straight to history if they only need the latest continuity result.
Transfer-ready
Queue work is mature enough for continuity handoff
Use this when the inquiry is closed or stable enough that remembered-return value should be preserved in a continuity record rather than living only in the queue timeline.
Draft-ready
A seeded draft exists but still needs operator review
This is useful when an operator wants create-mode context preserved from the inquiry while still keeping the continuity lane clearly marked as draft or watch-level.
Queue-only
Stay in the live inquiry lane for now
Not every inquiry needs continuity immediately. Leave purely active operational work in the queue until there is enough evidence of return value, referral readiness, or retention timing.

Glossary

Source anchor
The original inquiry reference and guest identity context that continuity work should keep tied to the live record.
Source posture
The queue or closing posture of the source inquiry, used to compare whether continuity is early, aligned, or already late to the handoff.
Continuity posture
The current loyalty lane status, usually continuity state plus stage, showing how mature the remembered-return work is.
Value signal
A compact commercial reading such as watch, promising, strong, or flagship that helps operators decide how much future-stay memory discipline is justified.
Latest outcome
The most recent recorded continuity result, used to tell whether the lane advanced, paused, was retained, or needs revisit work.
Latest packet
The most recent packet-readiness signal, such as no packet, watch, pending, or prepared. It helps operators judge how structured the next remembered-return move already is.
Next review
The next continuity checkpoint timing. Queue rows and loyalty rows surface this so operators can see revisit timing without opening deeper forms.
