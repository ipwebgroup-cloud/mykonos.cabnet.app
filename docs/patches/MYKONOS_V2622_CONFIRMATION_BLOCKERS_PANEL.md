# Mykonos Inquiry Platform v2.6.22 — Confirmation Blockers Panel

## Scope
Backend-only operator workspace refinement. No schema change, no public theme change, and no list filter change.

## What this patch adds
- a new **Confirmation Blockers** panel on the **Internal** tab
- a read-only scan-first surface for:
  - visible ownership blockers
  - stay-window blockers
  - brief/service blockers
  - contact-route blockers
  - follow-up checkpoint blockers
  - commercial-depth blockers

## Why this helps
As the operator workspace becomes denser, the next safe improvement is not more workflow logic but a cleaner read-only framing of what still blocks a confident next move. This panel keeps the operator focused on the few blockers that matter most before shaping a reply, proposal, or confirmation touch.
